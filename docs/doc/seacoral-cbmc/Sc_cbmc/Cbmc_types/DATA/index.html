The options for the CBMC tool. They will be passed as a JSON value to the tool with the option `--json-interface`. https://diffblue.github.io/cbmc//man/cbmc.html

Output of the tool. They are received as JSON values. All the following types are elements structuring CBMC's answer. These types should be abstract, except we need them for testing.

val add_tests : (Sc_lib.Ctypes_bridge.literal_binding * Basics.Ints.t) list -> cbmc_result -> cbmc_result

add_tests tests covered res Adds `tests` covering the properties whose identifiers are in `covered`. If the new tests do not cover new labels, the lazy suspension is not even forced and tests are not added to the result. If the new tests cover more labels and they are strictly less than the one already registered, replaces the tests in the result. Otherwise, adds the new tests and adds the covered labels.

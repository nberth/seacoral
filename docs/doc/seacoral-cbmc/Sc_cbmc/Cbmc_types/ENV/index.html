Module Cbmc_types.ENV

type size_detail =
  1. | AsFormal of {
    1. var : string;
    2. max : int;
    }
    (*

    Size is represented as a formal.

    *)
  2. | AsConst of int
    (*

    Size is known

    *)
type mem_details =
  1. | NotAPointer
    (*

    A simple value

    *)
  2. | Pointer of {
    1. first_size : size_detail;
    2. sub_pointers : size_detail list;
    }
    (*

    A pointer. Its size is specified by the `first_size` field while the `sub_pointers` represent the sub_arrays if the pointer is a multi dimensional array.

    *)

The size of an array/string. It can either be represented as a variable or a known constant.

type origin =
  1. | Global of {
    1. orig_name : string;
    2. harness_arg_name : string;
    }
  2. | Formal of string
  3. | Internal

The representation of a harness formal. Harness formals can either initialize globals, be an original formal or a fresh formal (for array size for example).

type var_environment = (Sc_lib.Cil_utils.var * mem_details * origin) Basics.StrMap.t

A map binding a variable name (as appearing in the harness) to its size and some information.
